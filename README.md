# Analyse de Base de Données SQL - Usine de Production

## Objectif
Ce projet présente des requêtes SQL que j'ai écrites pour analyser les données de production d'une usine.
C'est un exercice pour montrer mes compétences en SQL (SELECT, JOIN, GROUP BY, etc.).

## Contexte
Imaginons une usine avec 3 tables :
- **Machines** : Liste des machines (ID, Nom, Type).
- **Production** : Enregistrements de production (ID, Machine_ID, Quantité, Date).
- **Pannes** : Historique des pannes (ID, Machine_ID, Durée, Date).

L'objectif est de répondre à des questions métier comme :
- Quelle machine produit le plus ?
- Quelle machine tombe le plus en panne ?
- Quel est le taux de disponibilité des machines ?

## Compétences SQL utilisées
- `SELECT`, `WHERE`, `ORDER BY`
- `JOIN` (INNER JOIN, LEFT JOIN)
- `GROUP BY`, `HAVING`
- `COUNT`, `SUM`, `AVG`
- Sous-requêtes (Subqueries)

## Fichiers
- `queries.sql` : Toutes les requêtes SQL commentées.
- `schema.sql` : Création des tables (structure de la base).

## Pourquoi j'ai fait ça ?
En tant qu'ingénieur, je dois souvent analyser des données industrielles.
SQL est l'outil standard pour interroger des bases de données (capteurs, production, maintenance).
Ce projet me permet de montrer que je sais extraire et analyser des données.
